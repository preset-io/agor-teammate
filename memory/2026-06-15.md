## sc-111120: CSV upload modal shows warning highlight after selecting database

**Reported:** 2026-06-15 by Daniel Watania via #bug-reporting
- Slack thread: https://preset-io.slack.com/archives/C0AGRNNURGX/p1781515786622239
- Environment: 2cad (not reproducible on prod — regression)

**Story:** https://app.shortcut.com/preset/story/111120
**SC state:** → Implementing (500020185)
**Epic:** 101517 (Stabilize Master)

### Root Cause

`validateDatabase` in `superset-frontend/src/features/databases/UploadDataModel/index.tsx` (line 581) reads `currentDatabaseId` from a stale closure instead of using the `value` param Ant Design passes to it. Ant Design triggers the validator synchronously on `onChange`, before React re-renders with the updated state — so the validator sees `currentDatabaseId === 0` and rejects even though a valid database was just selected.

### Fix

Use `value?.value` in the validator instead of `currentDatabaseId`:
```ts
const validateDatabase = (_: any, value: { value: number; label: string } | null | undefined) => {
  if (!value?.value) {
    return Promise.reject(t('Selecting a database is required'));
  }
  return Promise.resolve();
};
```
