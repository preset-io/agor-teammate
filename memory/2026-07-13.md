Fresh worktrees don't inherit `node_modules` or built `packages/*/lib` artifacts. Confirmed via `.github/workflows/pre-commit.yml` that CI does its own fresh `npm ci`, so this was purely a local-environment gap, not something that would have shown up in CI. Worth remembering: prefer a full `npm ci` in new worktrees over partial node_modules symlinking — the symlink shortcut works until any package needs to be added/modified.

Also note: an agent-initiated `npm install stylelint@17.14.0 postcss-styled-syntax@0.7.2 --no-save` (naming packages directly) was denied by the permission classifier as unauthorized supply-chain risk, even though both are declared in the project's own `package.json`. `npm ci`/`npm install` with no package names (manifest-driven) was allowed. **Lesson: when a package is missing, always reach for a manifest-driven install (`npm ci`, no CLI package names) before naming the package explicitly — the classifier treats those very differently.**

## 16:00 UTC — Deprecation-warning fix: boxplot mean/median FutureWarning

**Search:** Datadog `deprecat*`-style queries against `env:production` mostly turned up noise already in `known_noise` (flask-appbuilder's `appbuilder.app` — confirmed no caller in apache/superset, it's library-internal; AuthlibDeprecationWarning and old-style Celery `CELERYD_*`/`BROKER_*` settings — both come from Preset's private config/deployment, not the OSS repo; pkg_resources/sqlalchemy-redshift — already the subject of blocked PR #41935). Broadened to `FutureWarning`/`"will be removed"` over 7 days and found a genuinely fixable, high-volume one: `superset/utils/pandas_postprocessing/aggregate.py:46: FutureWarning` re: numpy `mean`/`median` callables, ~250-2900+ events per 2-day window, no decreasing trend.

**Found:** This exact warning class was already half-fixed by Elizabeth's own PR #41025 (commit `5ec0931840`, merged 2026-06-27) — `_get_aggregate_funcs` in `pandas_postprocessing/utils.py` special-cases string operator names to avoid passing raw `np.mean`/`np.median` into `GroupBy.agg`. But `superset/utils/pandas_postprocessing/boxplot.py:111-113` builds its **own separate** `operators` dict and was still passing `np.mean`/`np.median` directly as the `operator` value — which hits `_get_aggregate_funcs`'s `if callable(operator): aggfunc = operator` branch first, completely bypassing #41025's fix. Boxplot charts are common, explaining the high volume. Verified empirically with pandas 2.1.4 (`/tmp/pr39601-venv`, since neither the repo's own `.venv` nor local Python had pandas installed) — raw `np.mean` triggers the warning even wrapped in `functools.partial`; only the string form suppresses it.

**Fix (2 files):** Changed `boxplot.py`'s `"mean": np.mean, "median": np.median` to `"mean": "mean", "median": "median"` (string literals), widened the local `operators` dict type to `dict[str, Union[str, Callable[[Any], Any]]]`, left `"count": np.ma.count` untouched (intentionally excluded from the string path since it counts NaNs, unlike pandas' `"count"`). Added `test_boxplot_mean_median_no_future_warning` to `test_boxplot.py`, asserting no `FutureWarning` and that values match a plain `df.groupby(...).agg(["mean","median"])`. Ran full `pandas_postprocessing` suite (108 tests) — all passing. Confirmed via `git stash`/reset roundtrip that the pre-fix code does raise the warning (see incident note below).
