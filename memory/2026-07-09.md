## a11y: RightMenu documentation/bug-report icon links missing aria-label (10:00 UTC)

**Issue found:** `superset-frontend/src/features/home/RightMenu.tsx` lines 753-764 and 770-781 (`superset-frontend/src/features/home/`) — the top-right navbar's documentation and bug-report links are icon-only `<StyledAnchor>` (`<a>`) elements with only a `title` attribute, no `aria-label`/`aria-labelledby`/visible text. `title` is not a reliable accessible name for assistive technology.

WCAG 2.1 SC 4.1.2 (Name, Role, Value) — Level A.

**Search process:** two Explore-agent rounds. First round surfaced `dashboard/components/IconButton.tsx` (shared component, multiple call sites) — ruled out because it's the exact component covered by already-open PR #41470 ("add aria-label to dashboard IconButton and unlabeled call sites"). Confirmed via `gh pr list --search "aria-label OR accessibility OR a11y"` (30 open PRs) and `gh pr list --search "RightMenu"` (none touching this file) that nothing else is in flight here. Also checked PR #39230 ("add accessible names to charts, status icons, and icon buttons", still OPEN, 13 files) — doesn't touch RightMenu.tsx either.

**Fix:** added `aria-label={navbarRight.documentation_text || t('Documentation')}` and `aria-label={navbarRight.bug_report_text || t('Report a bug')}`, reusing the exact same expression already used for each link's `title`, so any admin-configured custom text is respected. 2-line net diff, 1 file.
