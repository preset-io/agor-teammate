# 2026-06-13 — Deprecation Warning Fix Session

## Deprecation Warning Found

**Message:** `CDeprecationWarning: The 'CELERYBEAT_SCHEDULE' setting is deprecated and scheduled for removal in version 6.0.0.` (and `BROKER_POOL_LIMIT` same)
**Service:** superset (celery worker/beat)
**Source:** `celery/app/utils.py:203`
**Root cause:** `superset-shell/helm/superset/values.yaml` `celeryConfig` block and `helm/superset/config-templates/preset_config.py.tpl` used Celery 3.x/4.x uppercase config keys. These get injected into the production `CeleryConfig` class at deploy time. Celery 5.x warns on every process startup.

Other warnings found (not actioned today):
- `appbuilder.app is deprecated` — already fixed in PR #40876 (June 9), not yet deployed to all pods
- `AuthlibDeprecationWarning: authlib.jose` — open branch `aminghadersohi/ch107360/suppress-authlib-jose-warning` in superset-shell
- `Superset.explore_json deprecated` — intentional endpoint deprecation, clients still calling it; needs broader migration
- `pkg_resources deprecated` from sqlalchemy_redshift — third-party

## Fix

**Repo:** preset-io/superset-shell  
**Branch:** `fix/celery-deprecated-config-keys`  
**Commit:** `5b17470683`  
**PR:** https://github.com/preset-io/superset-shell/pull/4229

Changed three files:
- `helm/superset/config-templates/preset_config.py.tpl`: `CELERYBEAT_SCHEDULE` → `beat_schedule`
- `helm/superset/values.yaml`: all 13 uppercase `celeryConfig` keys → lowercase Celery 5.x names; deduplicated `CELERY_ENABLE_REMOTE_CONTROL` + `CELERY_WORKER_ENABLE_REMOTE_CONTROL` → single `worker_enable_remote_control`
- `docker/preset_config.py`: same key renames; removed `CELERY_TASK_PROTOCOL = 2` (default) and `CELERYD_LOG_LEVEL` (not a class config key in Celery 5.x)
