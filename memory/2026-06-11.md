- **Root cause:** authlib 1.3+ emits `AuthlibDeprecationWarning` (configured as "always") on every `authlib.jose` import. `fastmcp` imports it internally; the warning fires on every worker startup.

The production warning is from `superset-shell/preset/mcp/oauth_factory.py` (private repo, worktree `ch107360-suppress-authlib-jose-warning` exists but not yet merged). The OSS superset also imports from `authlib.jose.errors` in `mcp_service/` ŌĆö same class of bug, fixed in the OSS PR.

### Fix
Branch `fix/authlib-jose-deprecation-warning` off `origin/master` in OSS superset.

**First attempt (WRONG):** Wrapped `from authlib.jose.errors import ...` in `with warnings.catch_warnings():` blocks inside each file. Self-review caught this as a no-op: fastmcp imports `authlib.jose` earlier in the same import chain, so `authlib.jose` is already in `sys.modules` before the `with` block runs.

**Correct fix:**
- `superset/mcp_service/__init__.py`: Added module-level `warnings.filterwarnings("ignore", message=r"authlib\.jose module is deprecated")`. Python always runs `__init__.py` before any submodule import, so the filter is active before fastmcp triggers the warning. Works for both prod path and test imports.
- `superset/mcp_service/server.py`: Added authlib to `_suppress_third_party_warnings()` for defense in depth.
- Restored normal import positions in `jwt_verifier.py`, `mcp_config.py`, `test_jwt_verifier.py` (error classes stay from authlib, not joserfc, because fastmcp raises authlib exceptions).

Two commits: `4d8c3fa942` (wrong), `871c6db5b3` (correct). Both pre-commit hooks passed.

**Key lesson:** When using `warnings.catch_warnings()` to suppress a warning from a transitive import, you must place the suppression BEFORE the import that triggers the chain ŌĆö not around your own downstream import. `__init__.py` is the right place for package-level warning suppression.
