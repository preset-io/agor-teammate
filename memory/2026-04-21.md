## Unified shell+manager Agor env (infra, not a bug)

Long conversation with Max on how to fire up Manager + Superset-Shell under one `docker compose up` in Agor. Decision tree:
- Shell cannot run without Manager (JWT keys, workspace config) → only real variants are `manager-only` and `shell+manager`
- Dependency direction (shell → manager) = submodule direction → **submodule `preset-io/manager` INTO `superset-shell` at `./manager/`**. Parallel to existing `superset-oss` submodule, not nested.
- Rejected options: new "Hyperset" repo (unneeded ceremony), manager-as-pulled-image (requires per-SHA CI tags, worse for Agor isolation), submodule shell into manager (wrong dependency direction)
- Auto-bump manager@main on env start is default; `PINNED_MANAGER=1` for reproducibility, `MANAGER_REF=<sha>` for specific version. No cron needed.
- v1 runs two postgres containers side-by-side (shell's pgvector:14, manager's pg17 tmpfs). v2 can converge. ~400-600MB idle cost per worktree; not worth blocking v1 on.
- All host ports parameterized via env vars so Agor can assign unique offsets per worktree (follow `apache/superset` and `preset-io/agor` config patterns).
