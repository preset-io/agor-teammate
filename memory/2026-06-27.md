## a11y: IconTooltip aria-label propagation (second fix, 10:10 UTC)

**Issue found:** `IconTooltip` (`packages/superset-ui-core/src/components/IconTooltip/index.tsx`) renders a link-style `<Button>` containing only an icon child. The `tooltip` string prop was shown as a hover tooltip but was **never set as `aria-label`** on the button — screen readers announced the icon's auto-generated name (e.g. "eye") instead of the human-readable action (e.g. "Show SQL").

WCAG 2.1 SC 4.1.2 — Level A violation.

**Fix:** 1 insertion in `IconTooltip/index.tsx`:
- Add `aria-label={tooltip ?? undefined}` to the `Button`
- Covers all 3 call sites automatically (ShowSQL, ExploreCtasResultsButton, CollectionControl)

**Branch:** `fix/a11y-icon-tooltip-aria-label` on apache/superset  
**Commit:** `1a8621801c`  
**PR:** https://github.com/apache/superset/pull/41493

**Pre-commit hook notes:** Same stale `lib/` issue — patched `superset-ui-core/lib/components/Button/index.d.ts` to reflect forwardRef export (source has it, stale `.d.ts` declared it as a plain function with no ref support). Gitignored; CI passes from source.
