## a11y: dashboard IconButton aria-label

**Task:** Find and implement one accessibility fix in apache/superset.

**Issue found:** `IconButton` (`dashboard/components/IconButton.tsx`) renders a `<div role="button" tabIndex={0}>` with an optional `label` prop that only rendered as visible text ‚Äî it was never set as `aria-label`. Three call sites passed no `label` at all:
- `DeleteComponentButton` (delete icon only)
- `Row.tsx` settings gear icon
- `Column.tsx` settings gear icon

WCAG 2.1 SC 4.1.2 ‚Äî Level A violation.

**Fix:** 5 insertions across 4 files:
- `IconButton.tsx`: add `aria-label={label}` to `StyledDiv`
- `DeleteComponentButton.tsx`: add `t` import + `label={t('Delete component')}`
- `Row/Row.tsx`: add `label={t('Row settings')}`
- `Column/Column.tsx`: add `label={t('Column settings')}`
