## a11y: ActionButton aria-label (10:00 UTC)

**Issue found:** `ActionButton` (`packages/superset-ui-core/src/components/ActionButton/index.tsx`) renders a `<span role="button" tabIndex={0}>` containing only an icon. The `label` prop was used only for `data-test` and never set as `aria-label` — screen readers announced the icon's SVG content instead of the action name.

WCAG 2.1 SC 4.1.2 — Level A violation (Name, Role, Value).

**Before searching, checked:**
- memory/2026-06-27: Fixed IconButton and IconTooltip — ActionButton not touched
- In-flight PRs: #39230 and #40062 do not modify `ActionButton/index.tsx`

**Initial fix:** `aria-label={label}` — 1 insertion, commit `559087dfb9`

**Self-review findings:**

Session: `019f0daf-8320-7d17-8046-903257fb8ced`
URL: https://agor.sandbox.preset.zone/ui/s/019f0daf83207d1780469032/
Verdict: **CHANGES REQUIRED**

1. **[BLOCKING] `label` holds test IDs, not human-readable names.** Every call site checked uses `label` for internal identifiers (`'edit-action'`, `'delete-action'`) and `tooltip` for the translatable user-facing string (`t('Edit query')`). Using `aria-label={label}` would announce "edit-action, button" to screen readers.
2. **[NON-BLOCKING] Missing `onKeyDown` for Space key** on `role="button"` span — pre-existing gap, follow-up ticket.
3. **[NON-BLOCKING] Potential double-announcement** when tooltip present — acceptable, AT suppress when label=description.
4. **[INFORMATIONAL] No `disabled`/`aria-disabled` handling** — no current gap.

**Corrected fix:** `aria-label={typeof tooltip === 'string' ? tooltip : label}` — commit `8aa0faf806`
