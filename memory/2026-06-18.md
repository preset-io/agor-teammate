## SC-111157 — Forecast feature throwing error

**Ticket:** https://app.shortcut.com/preset/story/111157
**Customer:** Ippen Digital
**Priority:** P3

### What happened
Sophie asked me to investigate and fix SC-111157. The bug: forecast feature throws an error when using "Day" time grain on the "Vehicle Sales" dataset.

### Orchestrator investigation findings
- Forecast uses Facebook Prophet via `superset/utils/pandas_postprocessing/prophet.py`
- `PROPHET_TIME_GRAIN_MAP` in `utils.py` maps time grain ISO strings to pandas freq aliases
- Several aliases are deprecated in pandas 2.2+ (`"H"` → `"h"`, `"S"` → `"s"`, `"M"` → `"ME"`, `"Q"` → `"QE"`, `"A"` → `"YE"`)
- "D" for Day is NOT deprecated, so the pandas alias isn't the direct cause of a Day-specific error
- `_prophet_fit_and_predict` has NO try-except around `model.fit()` / `model.predict()` — any prophet exception becomes an unhandled 500
- No minimum data-point check before calling prophet (needs >= 2 non-NaN rows)
- The error screenshot is behind Shortcut auth, exact error unknown
- Could not reproduce locally (no running Superset env)
