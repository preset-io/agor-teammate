## Fix

**Repo:** apache/superset
**Branch:** `fix/query-object-groupby-deprecated-rename`
**Commit:** `e1be5090b8`

**Change:** In `superset/common/query_object_factory.py`, `QueryObjectFactory.create()` now normalises all `DEPRECATED_FIELDS` (imported from `query_object.py`) before any downstream processing. For each deprecated field in `**kwargs`, it pops the old name and ‚Äî if the value is truthy ‚Äî calls `setdefault` on the new name (so an explicit canonical key is not overwritten). This means `QueryObject._rename_deprecated_fields` never sees deprecated keys for the factory path, so the warning is not emitted.

Also added two new unit tests in `tests/unit_tests/common/test_query_object_factory.py`:
- `test_deprecated_groupby_renamed_to_columns` ‚Äî basic rename case
- `test_deprecated_groupby_does_not_overwrite_columns` ‚Äî canonical key wins
