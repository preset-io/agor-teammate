**Signal chosen:** `groupby` deprecated in `QueryObject._rename_deprecated_fields`. Fires in `env:production` from workspace `8a26d867`. Root cause: `Slice.get_query_context()` at `superset/models/slice.py:285` parses stored `query_context` JSON and passes it directly to `QueryContextFactory.create()`, bypassing the Marshmallow `ChartDataQueryObjectSchema.rename_deprecated_fields` `@post_load` hook. Charts saved with the old `"groupby"` key emit this warning on **every render** with no caller able to act on it.

## Fix

**Repo:** apache/superset
**Branch:** `fix/query-object-groupby-deprecated-rename`
**Commit:** `e1be5090b8`

**Change:** In `superset/common/query_object_factory.py`, `QueryObjectFactory.create()` now normalises all `DEPRECATED_FIELDS` (imported from `query_object.py`) before any downstream processing. For each deprecated field in `**kwargs`, it pops the old name and — if the value is truthy — calls `setdefault` on the new name (so an explicit canonical key is not overwritten). This means `QueryObject._rename_deprecated_fields` never sees deprecated keys for the factory path, so the warning is not emitted.

Also added two new unit tests in `tests/unit_tests/common/test_query_object_factory.py`:
- `test_deprecated_groupby_renamed_to_columns` — basic rename case
- `test_deprecated_groupby_does_not_overwrite_columns` — canonical key wins

Pre-commit hooks all passed.
