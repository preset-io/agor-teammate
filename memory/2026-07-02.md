## a11y: VizTypeGallery search clear icon aria-label (10:00 UTC)

**Issue found:** `VizTypeGallery` (`superset-frontend/src/explore/components/controls/VizTypeControl/VizTypeGallery.tsx:781`) renders a "clear search" icon (`Icons.CloseOutlined`) with `onClick={stopSearching}` in the search input's suffix, but no explicit `aria-label`. It fell back to the shared `BaseIcon` component's auto-generated label (derived from the icon's component name — `genAriaLabel("CloseOutlined")` → `"close"`), which is generic and doesn't describe what the control does in context (clears the search and returns to the previously active category).

WCAG 2.1 SC 4.1.2 — Level A violation (Name, Role, Value): the accessible name present was non-contextual/misleading, not truly absent.

**Note on the codebase's aria-label mechanism:** `BaseIcon.tsx` auto-generates a generic `aria-label` for every icon based on its component name, and sets `role="button"` when `onClick` is present. This means most "icon-only button missing aria-label" candidates in this repo already have *some* fallback name — the recurring fix pattern (this PR + #41470, #41503, #41619) is replacing that generic auto-label with an explicit, translated, contextual one at the call site. Confirmed via `BaseIcon.tsx` prop-spread order that an explicit `aria-label` passed by the caller overrides the auto-generated one (spread after the default).

**Investigated but ruled out** (to avoid duplicating in-flight work): checked `gh pr list --search "aria-label OR accessibility OR a11y"` (51 open PRs, incl. #39230-#39245 broad WCAG sweeps, #40062 role=button-antipattern) and 3 candidates from a broader sweep — `ControlHeader.tsx:103` (info icon) and `TableCatalog.tsx:71` (table-catalog remove icon) were also clean and unclaimed, picked `VizTypeGallery.tsx` as the smallest, most isolated single-call-site fix.

**Fix:** `aria-label={t('Clear search')}` on the icon — 3 line insertion. Commit `1aa99dbb6ae`. Only call site of this icon/handler pattern in the codebase (confirmed via self-review grep).
