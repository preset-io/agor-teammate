## Root Cause

`_get_aggregate_funcs` in `superset/utils/pandas_postprocessing/utils.py` was always passing `functools.partial(np_func)` to `GroupBy.agg()`, even when no keyword options were needed. pandas warns when it receives numpy callables it internally maps to its own SeriesGroupBy methods (`np.mean` → `SeriesGroupBy.mean`), because that mapping will be removed in a future pandas version.

Fix: when `options` is empty and the operator is in `_PANDAS_STRING_AGGREGATORS` (`max`, `mean`, `median`, `min`, `prod`, `std`, `sum`, `var`), pass the string name directly. This calls the same pandas method with no behavior change.

`"count"` intentionally excluded: `NUMPY_FUNCTIONS["count"]` maps to `np.ma.count` (NaN-inclusive, returns total row count), while pandas `SeriesGroupBy.count` excludes NaN — different semantics.
