### Root Cause

Six methods in `preset/security/manager.py` (superset-shell) were accessing `self.appbuilder.app.config["SECRET_KEY"]` instead of `current_app.config["SECRET_KEY"]`:
- `copilot_enabled`
- `copilot_model_id`
- `copilot_api_key`
- `copilot_provider`
- `copilot_key_source`
- `custom_system_prompt`

`get_ai_assist_extra` in the same class already used `current_app.config` correctly. apache/superset itself was already clean (fixed in PR #40876).

**Branch:** `fix/appbuilder-app-secret-key` in `preset-io/superset-shell`

**Change:** Replaced all 6 occurrences of `self.appbuilder.app.config["SECRET_KEY"]` with `current_app.config["SECRET_KEY"]`. No new imports needed — `current_app` already imported at file top. No behavior change.
