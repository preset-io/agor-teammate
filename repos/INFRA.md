# Preset Infrastructure Repos — Deep Dive

**Last updated:** 2026-03-16
**Source:** README.md and CLAUDE.md from each repo

---

## Overview

Preset has **23 infrastructure-related repos** spanning IaC, deployment pipelines, container images, monitoring, cloud deployments (SaaS, MPC, PCS), and DR.

### Cloud Deployment Models
- **SaaS** — Multi-tenant, Preset-managed K8s clusters (the main product)
- **MPC (Managed Private Cloud)** — Preset manages infra deployed in the *customer's* cloud (flywire, rivian, wealthsimple, etc.)
- **PCS (Private Cloud Superset)** — Customer self-manages; Preset provides Helm charts, Docker images, and tooling

## 1. The Terraform Stack (3-Tier Architecture + Legacy)

### `infra` — The Original Monolith (DEPRECATED but important to understand)
- **Purpose:** The original, all-in-one Terraform + modules repo. Now deprecated in favor of the 3-tier split.
- **Tech:** Terraform 0.12.x, AWS, Atlantis, SOPS + KMS, aws-okta
- **157 directories, 7 environments:** development, devops, master, production, qa, sandbox, staging, superset-public
- **Key module: `app-stack`** — THE central module that provisions a complete Superset cluster:
  - **EKS cluster** (VPC, subnets, worker ASGs, IAM)
  - **RDS PostgreSQL** (Superset metadata DB + examples DB, with Datadog monitoring user)
  - **ElastiCache Redis** (2 instances: one for Celery broker `ss`, one for cache `sc`)
  - **S3 buckets** (query results + thumbnails, encrypted, 15-day lifecycle)
  - **VPC peering** (to DevOps VPN VPC + DevOps EKS VPC)
  - **IAM** (Superset service user, RDS backup, Datadog, CI)
- **`global` module** — Canonical source of truth for:
  - All environment names: development, devops, master, production, staging, sandbox
  - DNS domains: `preset.io` (public), `preset.zone` (private), `preset.net`
  - App domains: `app-dev.preset.io`, `app-stg.preset.io`, `app.preset.io`, `app-sdx.preset.io`
  - VPC CIDR allocations: All under `10.0.0.0/8` (K8s) and `172.16.0.0/12` (general/VPN)
  - CIDR map: dev=10.0.0.0/18, staging=10.8.0.0/18, production=10.16.0.0/18, devops=10.25.0.0/18, sandbox=10.33.0.0/18
- **Why it matters:** Even though deprecated, this repo documents the original architecture. The `app-stack` module shows exactly what "a Superset cluster" means in infrastructure terms. The newer `terraform-live-envs` has its own `app_stack` module that evolved from this.

### terraform-modules-resources (Bottom Tier — DEPRECATED)
- **Purpose:** Atomic, single-purpose Terraform modules (one resource = one module)
- **Tech:** Terraform/HCL, AWS, GCP, Auth0, Datadog, FiveTran
- **Contents:** `aws/` (ACM, Athena, CloudFront, EFS, ElastiCache, Glue, GuardDuty, IAM, KMS, Lambda, Redshift, Route53, S3, SecretsMgr, SNS, SSM, VPC, WAF), `gcp/`, `auth0/`, `datadog/`, `fivetran/`
- **Status:** "Currently avoided" per terraform-live-envs README — services tier is used directly instead
- **No README exists**

### terraform-modules-services (Middle Tier — Active)
- **Purpose:** Composable service-level modules that wrap third-party Terraform registry modules with Preset-specific config
- **Tech:** Terraform/HCL, AWS, GCP, Go/Terratest, Ansible, GitHub Actions
- **Contents:** `aws/` (EKS, RDS, S3, VPC, ACM, ArgoCD, Aviatrix, CloudFront, Datadog, EFS, ElastiCache, IAM, KMS, Lambda, RabbitMQ, Redshift, Route53, SNS, SSM, WAF, private-link, segment-io), `gcp/` (GKE, GCS, VPC, Cloud SQL, BigQuery, IAM), `ansible/` (playbooks e.g. superset_postgres), `tests/aws/` (Terratest)
- **CI:** `terraform fmt --recursive --check` + `ansible-lint`
- **Standard file layout:** `main.tf`, `required-variables.tf`, `default-variables.tf`, `outputs.tf`, `versions.tf`, `data.tf`

### terraform-live-envs (Top Tier — Very Active)
- **Purpose:** Environment-specific infrastructure configurations. THE top-level entrypoint for all Terraform changes.
- **Tech:** Terraform/HCL, AWS, GCP, Atlantis (CI/CD), SOPS (secret encryption via AWS KMS), Ansible
- **Environments:** development (deprecated), staging, production, sandbox, devops, master, qa, superset-public
- **Contents:**
  ```
  environments/<env>/<cloud>/regions/<region>/<component>/
  modules/aws/app_stack/     # Complete Superset + Manager application stack
  modules/aws/infrastructure/  # VPC and networking
  modules/gcp/app_stack/     # GCP application stack
  cloudformation/            # CloudFormation templates for MPC
  ```
- **MPC clients:** flywire, koa, rivian, wealthsimple, hmetrix, beam, cargosense, ippen, komgo, workiva-dev, tencent, preset2
- **CI:** Atlantis auto-applies on merged PRs. `atlantis.yaml` is code-generated by `sync_atlantis_projects.sh`
- **DNS convention:** `<PRODUCT>-<SERVICE><N>-<EKS_NAME>.<GLOBAL_REGION>.<ENVIRONMENT>.<PRESET_DOMAIN>`

### terraform-provider-preset-manager
- **Purpose:** Custom Terraform provider wrapping the Preset Manager API for managing cluster and deployment resources via IaC
- **Tech:** Go 1.19, Terraform Plugin SDK v2, Terratest
- **Resources:** `manager_cluster`, `manager_deployment`
- **Data sources:** `manager_cluster`, `manager_deployment`

### vpc-peering
- **Purpose:** Reusable Terraform module for cross-account AWS VPC peering (three-provider model: root, requester, accepter)
- **Tech:** Terraform ~1.0.11, AWS ~3.72.0
- **Use case:** MPC customer VPC peering

```
terraform-modules-resources  ← DEPRECATED, rarely used directly
         |
terraform-modules-services   ← Composable service modules (EKS, RDS, etc.)
         |
terraform-live-envs          ← Per-environment configs, Atlantis CI/CD
         |
         ├── SaaS clusters (production-aws-us1a, eu5a, ap1a, etc.)
         └── MPC clusters (flywire, koa, rivian, wealthsimple, etc.)
```

## 2. GitOps & Deployment

### argocd
- **Purpose:** Kubernetes GitOps — defines what apps are deployed to each cluster using app-of-apps pattern
- **Tech:** ArgoCD, Kustomize, ArgoCD Vault Plugin (AVP), SOPS, Okta SSO
- **Contents:** `environments/` (devops, sandbox, staging, production), per-cluster YAML files, MPC client app files
- **ArgoCD instances:** `argocd.<env>.preset.zone`
- **Secrets:** AVP with SOPS backend (KMS-decrypted at apply time)

### helm
- **Purpose:** Helm chart library + per-cluster deployment configurations (chart versions, values, secrets)
- **Tech:** Helm, CircleCI, ChartMuseum (private registry), SOPS + KMS, Python (`helm_sync.py`)
- **Charts include:** base-cluster, core-services, karpenter, cert-manager, falco, falcon-sensor, vault, vault-secrets-consumer-webhook, datadog, kubecost, nginx-ingress, kong-ingress, aws-load-balancer-controller, argocd, jenkins, spinnaker, atlantis
- **Config structure:** `config/environments/<env>/<cloud>/<region>/<cluster>/<deployment>/` with `chart-info.yaml`, `values.yaml`, `secrets.yaml`
- **Deployment:** `helm_sync.py` auto-discovers and syncs on merge to master via CircleCI
- **Deployment order:** Controlled by `order` field in `chart-info.yaml`. `deleted: true` triggers uninstall.

### service-deploy-pipeline
- **Purpose:** Jenkins pipeline for deploying backend services to K8s via Helm ("A pile of potatoes")
- **Tech:** Jenkins/Groovy, Helm, AWS S3, PagerDuty, Slack
- **Deploys:** api-gateway, birdsai, cronjobs, db-backups, jira-webhooks, manager, platform-broker, preset-bot, cord-shell, pingpong-patrol
- **Promotion model:** SHA must be tagged `staging/deployed` before production deploy
- **Production deploys require explicit confirmation (locked gate)**
- **Slack:** `#deployments` channel
- **PagerDuty:** Maintenance windows suppressed during deploys

### ephemeral-env-pipeline
- **Purpose:** Jenkins pipeline for lifecycle of short-lived sandbox environments (spin-up + hourly cleanup)
- **Tech:** Jenkins/Groovy, K8s/kubectl, Helm
- **Deploys:** manager + superset into sandbox cluster
- **URLs:** `manager-{deployment_id}.app-sdx.preset.io`
- **Cleanup:** Hourly cron, configurable lifetime (default 1 day)

### release-maker
- **Purpose:** Python CLI that automates Superset release branch creation across superset-private + superset-shell repos
- **Tech:** Python 3.9 (Click CLI), GitHub API, Shortcut, TestRail, Slack
- **Release types:** Standard (from master), Hotfix Apache Fork, Hotfix Shell Only, Hotfix Shell + Fork
- **Version scheme:** Calendar-based (e.g., `2021.35.0`). `.0` = from master, higher patch = hotfix

```
release-maker → cuts release branch
    |
service-deploy-pipeline → deploys to staging → promotes to production
    |
ephemeral-env-pipeline → spins up sandbox for testing
    |
argocd → GitOps delivery to clusters
    |
helm → chart versions + per-cluster config
```

## 4. Cloud Deployment Models (MPC + PCS)

Preset has two private deployment models beyond SaaS:
- **MPC (Managed Private Cloud):** Preset manages infra inside the customer's own cloud account
- **PCS (Private Cloud Superset):** Customer self-manages; Preset provides artifacts (Helm charts, images, tooling)

### MPC Repos

#### mpc-init
- **Purpose:** Customer-facing tooling for granting Preset IAM access to customer's cloud
- **Tech:** Terraform (GCP IAM module), Shell scripts, AWS IAM YAML
- **Two options:** Terraform module (for IaC shops) or shell script (quick setup)
- **Both methods are idempotent**
- **Creates:** `PresetMPCAdminV2` custom role + `preset-mpc-sa` service account
- **After setup:** Preset manages infra via `terraform-live-envs` (MPC client dirs: flywire, koa, rivian, wealthsimple, hmetrix, beam, cargosense, ippen, komgo, workiva-dev, tencent, preset2)

### PCS Repos

#### preset-pcs
- **Purpose:** Central release registry for PCS product. Customers pull tagged releases (MAJOR.MINOR.PATCH.BUILD)
- **Tech:** Go Template (Helm charts), Docker, K8s
- **Contents:** `helm/superset/` (Chart.yaml, values.yaml, templates, bundled postgresql + redis sub-charts), `docker/`, `builds/`, `changelogs/`
- **Canonical source of truth for PCS versions**

#### pcs-superset-fips
- **Purpose:** FIPS 140-3 compliant Superset Docker images on Red Hat UBI 9
- **Tech:** Docker, RHEL UBI 9, Python 3.11 (compiled from source), OpenSSL FIPS provider
- **Two-tier build:** FIPS Python base image + full Superset image
- **Validation:** 6 compliance checks at startup; container refuses to start if any fail
- **Uses `superset-oss` submodule from preset-pcs releases**

#### pcs-ivanti
- **Purpose:** Chainguard-based FIPS Superset images specifically for the Ivanti customer
- **Tech:** Docker, Chainguard base, FIPS (SHA256)
- **Hard dependency:** preset-pcs v6.0.0.1+ (MD5 → SHA256 switch)

#### pcs-superset-shell
- **Purpose:** Shell wrapper for Superset deployment delivered to PCS customers
- **Status:** Minimal/thin repo — mostly customer-facing scaffolding

### How the deployment models connect
```
=== MPC Flow ===
mpc-init → customer grants Preset IAM access
    |
terraform-live-envs → Preset provisions cluster in CUSTOMER's cloud
    |
argocd + helm → Preset deploys Superset to customer cluster

=== PCS Flow ===
preset-pcs → customer pulls Helm chart + images
    |
pcs-superset-fips / pcs-ivanti → FIPS images if required
    |
pcs-superset-shell → customer deploys to their own infra
```

## 5. Operations & Monitoring

### devops-tools
- **Purpose:** Python CLI for common DevOps tasks + comprehensive CLAUDE.md for K8s cluster analysis
- **Tech:** Python 3.11 (Click), kubectl, k9s, AWS CLI, saml2aws, gcloud, Locust
- **CLI commands:** db-backups, file-upload, git, kube, workspace
- **Also contains:** `aws-nuke/`, `eks_upgrades/`, `locust-tests/`, `pg-10-14-migration/`
- **CLAUDE.md:** Remarkably detailed K8s analysis guide — documents all kubectl commands, Karpenter, namespace naming (`ws--<identifier>--main`), safety rules (read-only on production)

### cronjobs
- **Purpose:** Python/Click framework for K8s CronJobs managing AWS resources
- **Tech:** Python 3.11+, Click, boto3, Helm, Sentry, Datadog APM
- **Known jobs:** `cycle-asg-instances` (refreshes EC2 ASG instances by tag)
- **Uses internal `service-lib` from `pypi.devops.preset.zone`**
- **Concurrency policy:** `Forbid` (standard)

### celery-prometheus-exporter
- **Purpose:** Prometheus metrics exporter for Celery task queues (Redis + RabbitMQ)
- **Tech:** Python, Prometheus, Celery, Docker
- **Factory pattern:** Cleanly extensible broker monitoring
- **Supports Superset's async infrastructure (queries, reports, cache warming)**

### disaster-recovery-plans
- **Purpose:** DR procedures and runbooks for Preset's critical infrastructure
- **Tech:** Pure documentation (Markdown)
- **Scenarios:**
  | Scenario | RTO |
  |----------|-----|
  | Single workspace loss | 1 hour |
  | All workspaces / metadata DB | 2 hours |
  | Complete AWS region failure | 2 hours |
  | Single AZ failure | 30 minutes |
- **Contacts:** Daniel Gaspar (Tech Lead), Elizabeth Thompson (Eng Director)
- **Slack channels:** `#p0-p1`, `#p0-p1comms`

### qa-infra
- **Purpose:** Docker images for QA database services (Trino + Druid) on AWS Lightsail
- **Tech:** Docker, AWS Lightsail, Trino, Apache Druid
- **Note:** Lightsail (not EKS) used for simplicity/cost in QA

## Shared Patterns Across All Infra Repos

1. **SOPS + AWS KMS** — Secret encryption everywhere (Terraform, Helm, ArgoCD)
2. **Atlantis** — Terraform CI/CD with auto-apply on merge
3. **Okta SSO** — AWS/GCP auth via saml2aws / aws-okta
4. **Environment naming:** development (deprecated), staging, production, sandbox, devops
5. **MPC clients as first-class entities** — Each gets their own directories/files/clusters
6. **Jenkins (legacy) → CircleCI/GitHub Actions (modern)** — Migration in progress
7. **Internal PyPI** — `pypi.devops.preset.zone` for shared Python libs
8. **Karpenter** — Node provisioning with `core=true:NoSchedule` taint for infra nodes
9. **K8s namespace convention:** `ws--<identifier>--main` for Superset workspaces

## What "A Superset Cluster" Means (from `infra/app-stack`)

Each Superset cluster (SaaS or MPC) provisions:

| Component | Resource | Details |
|-----------|----------|---------|
| Compute | EKS cluster | VPC, 3 AZs, worker ASGs, IAM |
| Metadata DB | RDS PostgreSQL | `superset` database |
| Examples DB | RDS PostgreSQL | `examples` database (sample data) |
| Broker | ElastiCache Redis | `ss` — Celery broker for async tasks |
| Cache | ElastiCache Redis | `sc` — Result/filter cache |
| Storage | S3 (encrypted) | Query results (15-day expiry) |
| Storage | S3 (encrypted) | Thumbnails (15-day expiry) |
| Networking | VPC peering | To DevOps VPN + DevOps EKS |
| Monitoring | Datadog users | On both Postgres instances |
| DNS | Route53 | Public + private zones per env |
| IAM | Superset user | S3 access, service permissions |

This is what `app-stack` in both `infra` (legacy) and `terraform-live-envs` (current) provision.
